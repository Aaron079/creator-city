# Creator City 管理员运营 SOP

**版本：** 第一版（2026-06-16）  
**适用对象：** Creator City 管理员  
**访问入口：** `/admin`（需要 ADMIN role）

---

## 安全边界（必读）

管理员操作有严格边界，以下内容**绝对禁止**：

- ❌ 不查看、不索要、不存储用户的 Provider API Key（平台 API 已在服务端加密，管理面板不展示 Key 明文）
- ❌ 不操作真实支付退款（第一版无自动退款，积分支付关闭）
- ❌ 不发放平台积分（credits 功能第一版对普通用户关闭）
- ❌ 不审批未经线下核实付款的会员订单（先确认收款，再审批）
- ❌ 不直接写生产数据库
- ❌ 不在管理面板执行任何不可逆操作前二次确认

| 页面 | 路径 | 说明 |
|---|---|---|
| 管理员总览 | `/admin` | 入口页，各功能链接 |

## 日常操作 SOP

### 1. 审核会员订单

**触发条件：** 用户提交了会员开通申请（在 `/account/membership` 提交）

**步骤：**
1. 访问 `/admin/membership`
2. 查看 PENDING 状态的订单
3. **离开面板，核实线下收款**（银行转账记录、付款截图等）
4. 核实成功后，回到面板点击「审批通过」
5. 系统自动将用户 `membershipActive` 设为 `true`，有效期 30 天
6. 如未收到付款或信息有误，点击「拒绝」并备注原因

**审批注意事项：**
- 只有已确认收款才审批
- 不审批 QA/测试账号（除非是专用的 QA 订单）
- 审批后系统自动生效，无需手动改 DB

---

### 2. 查看用户会员状态

**步骤：**
1. 访问 `/admin/users`
2. 表格中查看「会员」列（绿色 = 活跃会员，灰色 = 未开通）
3. 查看「到期时间」列确认有效期
4. 查看「API 账户」列了解用户绑定了几个 Provider 账户

**用途：** 排查用户反馈"会员状态不对"时，用此页面确认。

---

### 3. 查看 Marketplace 合作意向

**步骤：**
1. 访问 `/admin/marketplace`
2. 点击「合作意向」tab
3. 查看买方/卖方沟通记录、意向状态

**重要：** 管理员只读，不在后台代替用户执行意向操作（respond/reject/close）。

---

### 4. 处理用户反馈

**接收渠道：** 用户通过 `/feedback` 页面提交的反馈，或直接联系管理员邮箱

**处理步骤：**
1. 阅读反馈内容（问题描述 + 账号邮箱 + 截图）
2. 在 `/admin/users` 查询该邮箱确认账号状态
3. 如是会员问题，在 `/admin/membership` 查看相关订单
4. 如是 Marketplace 问题，在 `/admin/marketplace` 查看相关 listing / 意向
5. 回复用户（当前无内置消息功能，请通过邮件或其他渠道回复）

---

### 5. 处理已知限制情况

| 用户诉求 | 管理员操作 |
|---|---|
| 想要退款 | 第一版无自动退款，按政策说明"不退款"；特殊情况 escalate |
| 想要积分 | 第一版积分对普通用户关闭，解释 BYOK 模式 |
| 视频 BYOK 不可用 | 确认用户凭证已存，告知接入中 |
| 会员审批慢 | 确认是否已收款，未收款不审批 |
| API Key 泄漏 | 提示用户去 Provider 控制台吊销，在平台删除账户重新添加 |

## 敏感操作检查清单

每次执行审批 / 拒绝操作前：

- [ ] 已核实线下付款记录
- [ ] 订单邮箱与付款人信息一致
- [ ] 不是 QA/测试订单（测试订单标注在备注中）
- [ ] 确认 membershipExpiresAt 将在合理范围内（30 天）
- [ ] 不查看、不索要用户 API Key

---

## 不在管理员职责范围内

以下操作不属于运营 SOP，如有需要需开发介入：

- 修改 Prisma schema 或数据库 migration
- 手动写入生产数据库
- 修改 Provider adapter 或 generate routes
- 修改 env 配置
- 修改 cn-executor 执行器
- NFT / wallet / on-chain 操作（第一版不做）
