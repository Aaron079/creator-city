# Creator City — Agent Skill Architecture

## 为什么工具必须是 Agent，而不是按钮

Creator City 的定位是 AI 导演工作台，不是参数调节面板。

一个按钮只能给 prompt 加几个英文词。一个 Agent 能理解当前素材、判断问题所在、给出专业解释、真实作用于图片/视频、输出可验收的结果。

**按钮**：用户感受不到价值，因为他们看不到改变，也说不清为什么要点这个按钮。

**Agent**：用户得到一个新版本的资产、一份分析报告、或一个可复用的 preset。他们知道这个 Agent 做了什么，为什么这么做。

用按钮堆 UI 是在模仿能力。真正的护城河在于：**能让普通创作者以专业电影人的视角处理每一帧画面**。

---

## Agent 标准结构

每个 Skill / Agent 必须完整实现以下六个维度：

```
┌─────────────────────────────────────────────────────────────┐
│  IDENTITY     明确身份：这个 Agent 是谁，解决什么问题        │
│  INPUT        明确输入：接收什么数据/素材                   │
│  ANALYSIS     明确分析：能解释为什么这么处理                │
│  ACTION       明确执行：真实作用于已有图片/视频或生成新资产  │
│  OUTPUT       明确输出：新节点、新图片、新视频、参数报告     │
│  PERSISTENCE  明确持久：结果保存到画布节点，可复用           │
└─────────────────────────────────────────────────────────────┘
```

### IDENTITY 身份
- 这个 Agent 的专业角色是什么？（构图师、色彩师、运镜导演……）
- 它解决的核心问题是什么？
- 在真实电影工业中对应哪个工种？

### INPUT 输入
- 当前画布节点的图片 URL / 视频 URL
- 用户描述的意图（prompt 或自然语言指令）
- 节点 metadata（比例、时长、生成参数）
- 项目风格 bible（如果存在）
- 用户在 UI 上的选择（可选，不强制）

### ANALYSIS 分析
- Agent 必须能输出一段可读的分析：为什么这张图构图有问题、为什么这个颜色不对、为什么运镜需要调整
- 分析结果显示在节点上，让用户看得懂
- 分析是可存储的，下次打开项目仍然可见

### ACTION 执行
两种执行模式：

**Mode A — 非生成式执行**（首选，成本低，速度快）
- CSS filter / transform 实时预览（客户端，零 API 调用）
- 裁切/重构图（Canvas API，客户端）
- 视频帧提取 + 叠加（客户端）
- 基于现有资产 URL 的参数调整

**Mode B — 生成式执行**（需要 API，产生新资产）
- 用增强后的 prompt 重新调用生成 API，产生新版本节点
- 新版本与原版本并排显示（diff 模式）
- 用户选择保留哪个版本

Agent 必须明确标注当前是 Mode A 还是 Mode B，以及是否消耗 token/额度。

### OUTPUT 输出
- 新画布节点（可选）
- 更新后的图片/视频 URL
- 参数报告（JSON，存入 metadataJson）
- 可复用的 preset（供其他节点一键应用）
- 人类可读的处理说明（显示在节点 tooltip 或侧边栏）

### PERSISTENCE 持久化
- 执行结果保存到节点 `metadataJson.agentResults[agentId]`
- 下次加载画布仍然可见此 Agent 处理过的痕迹
- preset 可以导出为 JSON，分享给其他创作者

## 第一批 Agent 列表

### 1. Composition Agent / 构图裁切 Agent

**身份**：构图指导，专注于画面的视觉张力、主体位置、三分法、黄金比例。

**实际用途**：
- 分析当前图片的构图问题（主体偏移、地平线歪斜、留白不均）
- 建议裁切方案（三分法、中心构图、黄金螺旋）
- 预览多个裁切比例

**市场价值**：
- 小红书/抖音创作者不懂构图
- 摄影助手角色，帮助普通人拍出专业感
- 可作为独立付费 skill 销售

**用户如何看见它**：
- 点击图片节点 → 选择"构图分析" → 叠加三分法网格 → 看到高亮的构图问题 → 选择裁切方案 → 一键应用

**如何真实作用于已有图片/视频**：
- Mode A：Canvas API 客户端裁切，不消耗任何 API 额度
- Mode B：将裁切参数加入 prompt，重新生成更精准的构图版本

**是否需要新资产**：Mode A 不需要，Mode B 生成新节点并行显示

**护城河**：构图规则是专业知识，普通 AI 工具不做这层分析

---

### 2. Camera Motion Agent / 运镜 Agent

**身份**：摄影机运动导演，专注于镜头运动语言（推、拉、摇、移、跟拍）。

**实际用途**：
- 对已有静止图片添加动态感预览（CSS keyframe 动画）
- 对已有视频分析运镜类型，建议优化
- 为下一次视频生成提供精确的运镜 prompt

**市场价值**：
- 视频创作者不懂运镜语言
- 能帮视频节点实时预览"推镜感"而不消耗 API

**用户如何看见它**：
- 点击视频节点 → 选择"运镜分析" → 看到当前运镜类型标签 → 选择推镜/拉镜预览 → CSS 动画实时演示 → 确认后加入下次生成 prompt

**如何真实作用于已有图片/视频**：
- Mode A：CSS keyframe（推镜 = scale 1→1.08，拉镜 = scale 1.08→1，摇镜 = translateX），实时、零成本
- Mode B：生成带运镜指令的新视频

**护城河**：把电影运镜语言翻译成可执行的创作操作，这是专业门槛

---

### 3. Colorist Agent / 色彩师 Agent

**身份**：色彩调级师，专注于色调、饱和度、对比度、色彩风格（冷暖、胶片感）。

**实际用途**：
- 对已有图片/视频实时调色预览（CSS filter，零 API）
- 分析色彩风格与项目整体色调的一致性
- 输出可复用的调色 preset

**市场价值**：
- 调色是后期制作的核心步骤，普通工具不做
- preset 可以成为付费模板

**用户如何看见它**：
- 点击节点 → 选择"色彩分析" → 看到当前色温/饱和度/对比度数值 → 滑块实时调整（CSS filter） → 确认 → 结果写入节点

**如何真实作用于已有图片/视频**：
- Mode A：CSS filter，实时，零成本（`saturate()`, `brightness()`, `contrast()`, `hue-rotate()`, `sepia()`）
- Mode B：将调色参数编译为精确 prompt，重新生成

**护城河**：把专业调色 LUT 翻译为可执行的 AI prompt + CSS 预览

---

### 4. Lighting Agent / 灯光 Agent

**身份**：灯光指导，专注于光线方向、光质、戏剧性灯光（逆光、伦勃朗、霓虹）。

**实际用途**：
- 分析当前图片的光线方向和质量
- 叠加灯光效果预览（CSS 渐变 overlay）
- 建议为下次生成修改灯光 prompt

**市场价值**：
- 灯光是摄影/影视中最难掌握的技能之一
- 能让普通创作者以灯光师视角审视素材

**用户如何看见它**：
- 点击图片节点 → 选择"灯光分析" → 叠加光线方向箭头 → 选择"逆光"样式 → 叠加金边渐变预览 → 确认后写入 prompt

**如何真实作用于已有图片/视频**：
- Mode A：CSS gradient overlay，零成本，实时预览
- Mode B：生成带精确灯光指令的新版本

---

### 5. Cinematographer Agent / 摄影指导 Agent

**身份**：综合摄影指导，整合焦距、光圈、镜头类型、景深为一个完整决策。

**实际用途**：
- 提供"85mm + f/1.4 + 逆光"这种组合决策
- 分析当前图片适合什么焦距风格
- 生成完整的镜头语言报告

**市场价值**：
- 这是最专业的 Agent，面向电影/广告创作者
- 可作为高级付费 skill

**如何真实作用于已有图片/视频**：
- Mode A：CSS 模拟景深（背景模糊度），结合 overlay
- Mode B：完整镜头参数注入生成 prompt，产生新版本节点

---

### 6. Storyboard Agent / 分镜 Agent

**身份**：分镜导演，专注于多节点的叙事顺序、镜头衔接、情绪节奏。

**实际用途**：
- 分析多个节点的视觉关系，判断叙事连贯性
- 建议调整节点顺序
- 为整个画布生成分镜说明书

**市场价值**：
- 视频创作者需要分镜能力
- 输出物可以直接给导演/制作团队

**如何真实作用于已有图片/视频**：
- Mode A：分析现有节点内容，输出文字报告
- Mode B：补充缺失的中间帧/过渡节点

## 第一优先级建议：先做 Composition Agent

**理由**：

1. **零 API 成本**：纯客户端 Canvas API 裁切，不消耗任何生成额度
2. **最容易验收**：用户直接看到裁切结果，不需要等待生成
3. **不依赖后端**：不改动生成链路、不改动 canvas save 路由
4. **真实可执行**：每次点击都有真实的、可见的素材变化
5. **立即有专业价值**：构图分析是电影人日常使用的技能

**最小可行版本（V1）**：
- 点击图片节点上的"构图"按钮
- 叠加三分法/中心构图网格（SVG overlay）
- 显示主体位置分析（基于亮度重心计算）
- 提供 3 个裁切预设（16:9 / 9:16 / 1:1）
- 点击任意预设，Canvas API 裁切，更新节点 resultImageUrl
- 结果写入节点 metadataJson.compositionAnalysis

**技术路径**：
```
用户点击"构图" 
  → 加载图片到 <canvas>
  → 计算亮度重心（brightness centroid）
  → 渲染网格 overlay
  → 用户选择裁切方案
  → canvas.toBlob() → 上传为新资产
  → 更新节点 resultImageUrl → 触发 canvas save
```

---

## 禁止再做的事

- 只选 chip 但不改变任何素材的假控制
- 只给 prompt 加几个英文词的按钮
- 不可验收的"导演感"UI
- 没有输出物的分析
- 不能独立销售的工具

每个 Agent 上线前必须能回答：
> "用户点击之后，画布上具体哪个东西改变了？"

如果回答不了，这个功能不应该上线。
